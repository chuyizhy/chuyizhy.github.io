一、GPIO初始化。

代码如下（示例）：

1、GPIO_InitTypeDef GPIO_InitStructure;

定义一个 GPIO 初始化结构体，用来配置 GPIO 的 模式、速度、引脚号 等参数。

2、GPIO_InitStructure.GPIO_Mode = GPIO_Mode_Out_PP;

GPIO_Mode_Out_PP 表示 推挽输出模式（Push-Pull Output）
推挽输出特点：
可以输出高电平或低电平
能够直接驱动 LED、电机等负载
另一种常用模式是 开漏输出（Open-Drain）只有低电平有效

3、GPIO_InitStructure.GPIO_Pin = GPIO_Pin_12;

指定要初始化的引脚是 PB12
可以同时初始化多个引脚，例如：
GPIO_InitStructure.GPIO_Pin = GPIO_Pin_12 | GPIO_Pin_13;

4、GPIO_InitStructure.GPIO_Speed = GPIO_Speed_50MHz;

配置 输出速度
50MHz 并不是引脚真正频率，而是 GPIO 输出驱动速度等级
影响电平上升/下降的速度，常用值：
10MHz
2MHz
50MHz（最快）

5、GPIO_Init(GPIOB, &GPIO_InitStructure);

GPIOB：要初始化的 GPIO 端口
&GPIO_InitStructure：传入配置结构体
功能：根据结构体的配置，将 PB12 设置为推挽输出，50MHz 输出速度

6、PA15、PB3、PB4默认JTAG调试端口不能直接使用

二、 GPIO输出函数

代码如下（示例）：

1. GPIO_SetBits(GPIO_TypeDef* GPIOx, uint16_t GPIO_Pin);

功能：将指定引脚 置高电平（逻辑 1）
参数：
GPIOx：GPIO 端口，如 GPIOA, GPIOB
GPIO_Pin：要操作的引脚，宏定义如 GPIO_Pin_0, GPIO_Pin_12
可以同时操作多个引脚，如 GPIO_Pin_0 | GPIO_Pin_1

2. GPIO_ResetBits(GPIO_TypeDef* GPIOx, uint16_t GPIO_Pin);

功能：将指定引脚 置低电平（逻辑 0）

3. GPIO_WriteBit(GPIO_TypeDef* GPIOx, uint16_t GPIO_Pin, BitAction BitVal);

功能：根据 BitVal 设置指定引脚的状态
BitVal 可以是：
Bit_SET → 输出高电平
Bit_RESET → 输出低电平
参数：
GPIOx：端口
GPIO_Pin：单个引脚
BitVal：状态值

4. void GPIO_Write(GPIO_TypeDef* GPIOx, uint16_t PortVal);

功能：直接写入 整个 GPIO 端口的输出寄存器，一次性更新所有引脚
参数：
GPIOx：端口
PortVal：16 位数值，每一位对应一个引脚状态
1 → 高电平
0 → 低电平

三、 GPIO输入函数

代码如下（示例）：

1. uint8_t GPIO_ReadInputDataBit(GPIO_TypeDef* GPIOx, uint16_t GPIO_Pin);

功能：
读取 某一个引脚的实际输入电平。
返回值：
1（Bit_SET） → 引脚为高电平
0（Bit_RESET） → 引脚为低电平
说明：
读取的是 IDR（Input Data Register）
不论该引脚被配置为 输入还是输出模式，都能读到“引脚上的真实电平”

2. uint16_t GPIO_ReadInputData(GPIO_TypeDef* GPIOx);

功能：
一次性读取 整个 GPIO 端口 16 个引脚的输入状态。
返回值：
16 位数据，每一位对应一个引脚的输入电平
说明：
读取的是 IDR（Input Data Register）

3. uint8_t GPIO_ReadOutputDataBit(GPIO_TypeDef* GPIOx, uint16_t GPIO_Pin);

功能：
读取 某一个引脚被“软件设置”的输出状态。
返回值：
1 → 软件设置为高电平
0 → 软件设置为低电平
说明：
读取的是 ODR（Output Data Register）
反映的是“你写了什么”，不一定等于引脚真实电平
例如外部短路、过载时，实际电平可能不同

4. uint16_t GPIO_ReadOutputData(GPIO_TypeDef* GPIOx);

功能：
读取 整个 GPIO 端口 16 位的输出寄存器值。
说明：
读取的是 ODR（Output Data Register）

IDR → 引脚现在“是多少”
ODR → 你“让它是多少”
